Routing

To handle the navigation from one view to the next, you use Angular router. The router enables navigation by interpreting a browser URL as an instruction to change the view. "ng new" creates a new workspace and project with an AppRoutingMoudule, which is an NgModule.

base href
Make sure <base href="/"> exists in the<head> of your index.html file. This assumes that the "app" folder is the application root, and uses "/"

AppRoutingModule
AppRoutingModule should be imported in the AppModule to exist in the imports array.

Template with routerLink and router-outlet
This nav gives you links to click, which tells the router which route to use (defined in the routes constant in AppRoutingModule)
<nav>
  <ul>
    <li><a routerLink="/first-component" routerLinkActive="active">First Component</a></li>
    <li><a routerLink="/second-component" routerLinkActive="active">Second Component</a></li>
  </ul>
</nav>

The routed views render in the <router-outlet>.
<router-outlet></router-outlet>

Route Order
The order of routes is important because the Router uses a first-match wins strategy when matching routes, so more specific routes should be placed above less specific routes.

Route Guard
The following command will generate a route guard and a test file (login.guard.ts and login.guard.spec.ts).
ng g guard login

The following are different types of the route guard. These are the interfaces that can be implemented in the guard.
  1. CanActivate
  2. CanActivateChild
  3. CanDeactivate
  4. Resolve
  5. CanLoad
