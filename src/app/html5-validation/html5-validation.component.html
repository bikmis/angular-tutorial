HTML5 Validation


HTML5 added new semantic types for the <input> element and contraint validation to ease the work of checking the form content on the client side. Basic constraints can be checked without the need for JavaScript by setting new attributes and complex constraints can be tested using Constraint Validation API. For details, check Constraint Validation and Client-side Form Validation. HTML5 constraint validation doesn't remove the need for validation on the server side.

if "novalidate" attribute is set on the <form> element, interactive validation of the constraints doesn't happen. Calling the submit() method on the <form> doesn't trigger a constraint validation and the method sends the form data to the server even if it doesn't satisfy the constraints. Call a click() method on a submit button instead.

":required", ":optional" pseudo classes are generated to represent required and optional fields. ":valid" and ":invalid" pseudo classes are used to represent <input> elements whose content valiates and fails to validate respectively according to the input's type setting.

Example:
  1. <input type="URL">

    The value must be an absolute URL.
  2. <input type="email">

    The value must be a syntactically valid email address.
  3. <input id="emailAddress" type="email" multiple>

    The input can accept multiple comma-separated email addresses.
  4. <input id="firstName" type="text" pattern="(\S)+">

    The value must match the pattern's regex with input types (text, search, url, tel, email, password). The regex "(\S)+" matches one or more character (other than a white space). + means one or more of the preceding character.
  5. <input type="date" min="2019-12-25" step="1">
  6. <input type="month" min="2019-12" step="12">
  7. <input type="week" min="2019-W23" step="">
  8. <input type="time" min="09:00" step="900">
  9. <input type="datetime-local" min="2019-12-25T19:30">
  10. <input type="number" min="0" step="5" max="20">

  11. <input type="range" min="60" step="5" max="100">

    The input will slide from 60 to 100 with a difference of 5 between two numbers one following the other.
  12. <input type="date" max="2019-12-25" step="1">
  13. <input type="month" max="2019-12" step="12">
  14. <input type="week" max="2019-W23" step="">
  15. <input type="time" max="17:00" step="900">
  16. <input type="datetime-local" max="2019-12-25T23:59">
  17. <input type="text" id="username" name="username" required>
  18. <input type="text" minlength="5" id="fruit">
  19. <input type="password" maxlength="4"/>
