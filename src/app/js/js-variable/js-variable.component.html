let(block-scoped)

A block scope means the scope within curly braces. Type the following code inside of the script tag at the bottom of the body tag. Run myFuncLet() or window.myFuncLet() in Console. This function throws "Uncaught ReferenceError: x is not defined". The error comes from console.log(x). x is not visible outside of "if" block. let variables are used in "for" loops, "if" blocks etc. "const" and "let" are stored in the global environment's declarative environment record. Even if "const" and "let" variables are declared at the global level, they are not available on the window object.
function myFuncLet() {{'{'}}
  if(true){{'{'}}
    let x = 7;
  }
  console.log(x);
};

The error comes from console.log(x). x is not visible outside of "if" block. "const" and "let" are stored in the global environment's declarative environment record. Even if "const" and "let" variables are declared at the global level, they are not available on the window object.

};

A constant cannot be redefined/redclared. In the below code, x is defined twice - the JavaScript engine will throw the following error: "Uncaught SyntaxError: Identifier 'x' has already been declared." and the function won't be available. function myFuncConst2() {{'{'}}
  if(true){{'{'}}
    const x = 7;
    const x = 8;
  }
  console.log(x);
};

A constant cannot be reassigned. In the below code, x is assigned twice: first with 7 and then 8. When you run the function, it will throw the following error: "Uncaught TypeError: Assignment to constant variable."

  if(true){{'{'}}
    const x = 7;
    x = 8;
  }
  console.log(x);
};

Declarations of Variables And Functions Get Hoisted

Only delclarations are hoisted, but not initializations. The variable and function declarations are put into memory during the compile phase. x is used in console.log(x) before it is declared in the function. x is not global here, it is only scoped within the function. The function catName(name) is called before it is declared. This is ok because declaratins of variables and functions get hoisted, but initializations don't get hoisted.

  x = 4;
  console.log(x);
